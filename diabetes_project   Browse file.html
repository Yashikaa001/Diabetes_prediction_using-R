What is this step and why do we do it?

Before we start any analysis or model building, we must first load the necessary R libraries (packages) and import the dataset into our working environment. Without loading these libraries, we would have to write hundreds of lines of code from scratch.

Step 02 — Dataset Overview & Preprocessing

Understanding and Preparing the Data

About the Dataset

We are using the Pima Indian Diabetes Dataset, which contains medical records of 768 female patients. Each row represents one patient, and there are 8 input features (predictor variables) and 1 output column called Outcome. The Outcome column is our target variable — it contains either 0 (patient does NOT have diabetes) or 1 (patient DOES have diabetes). This makes it a binary classification problem.

What is Preprocessing and why is it important?

Raw data from the real world is almost never perfectly clean or model-ready. Preprocessing is the process of transforming raw data into a clean, model-ready format. In this project, preprocessing involves three key tasks:

Checking for missing values — We use colSums(is.na(data)) to find any NULL or NA entries. In this dataset, no missing values were found.
Feature Scaling (Z-score Normalization) — Transforms all features to the same scale (mean = 0, SD = 1) using the scale() function.
Train-Test Split (70/30) — 537 patients for training, 231 for testing. We use set.seed(123) for reproducibility.

Step 04 — Building the Model

Training the Logistic Regression Model

What is Logistic Regression?

Logistic Regression is a supervised machine learning algorithm used for binary classification problems — problems where the output is one of only two categories, like Yes/No or in our case, Diabetic (1) / Non-Diabetic (0). Unlike regular (linear) regression which predicts a continuous number, logistic regression predicts a probability between 0 and 1 using a mathematical function called the Sigmoid function. If the predicted probability is greater than 0.5, the patient is classified as diabetic.

How is the model built in R?

In R, we use the glm() function — which stands for Generalized Linear Model. We pass family = binomial to tell R that we want logistic regression. The model learns the relationship between all 8 features and the Outcome by adjusting internal values called coefficients. A positive coefficient means that feature increases diabetes risk, while a negative coefficient decreases it.

Glucose has the highest significance (***) with a positive coefficient. Diabetes Pedigree Function (family history) and BMI are also strongly significant predictors.

Step 06 — Making Predictions

Predicting Diabetes Risk for a New Patient

How does prediction work on new patient data?

After training and evaluating the model, we can now use it to predict the diabetes risk for any new patient whose outcome is not yet known. We define a custom R function called predict_diabetes() that accepts the same 8 clinical values as input parameters and returns a prediction. Internally, it creates a single-row data frame from those inputs and passes it to predict(log_model, newdata = input, type = "response"). This returns a probability between 0 and 1, converted to a binary label using the 0.5 threshold.

For example: a patient with Glucose = 148, BMI = 33.6, Age = 50, and pedigree = 0.627 gets predicted as Diabetic, because the computed probability is above 50%.

06_predict.R
# ── Step 6: Predict for a New Patient ────────────────────
predict_diabetes <- function(Pregnancies, Glucose, BloodPressure,
                              SkinThickness, Insulin, BMI,
                              DiabetesPedigreeFunction, Age) {
  input <- data.frame(
    Pregnancies = Pregnancies, Glucose = Glucose,
    BloodPressure = BloodPressure, SkinThickness = SkinThickness,
    Insulin = Insulin, BMI = BMI,
    DiabetesPedigreeFunction = DiabetesPedigreeFunction, Age = Age
  )
  prob <- predict(log_model, newdata = input, type = "response")
  if (prob > 0.5) cat("Prediction: DIABETIC (prob =", round(prob, 3), ")\n")
  else           cat("Prediction: NON-DIABETIC (prob =", round(prob, 3), ")\n")
}

# Example: Patient from row #1 of dataset
predict_diabetes(6, 148, 72, 35, 0, 33.6, 0.627, 50)

Conclusion

What We Learned from this Project

Summary of findings and outcomes

This project successfully demonstrates a complete end-to-end machine learning pipeline in R for predicting diabetes. Starting from raw clinical data, we arrived at a working prediction system covering every stage — from data loading and cleaning to live patient prediction.

The Logistic Regression model achieved 77.8% accuracy on the 231-patient test set — significantly better than a naive baseline (65%). The model's 94% specificity makes it very reliable at confirming healthy patients. However, the 58% recall reveals that the model still misses about 42% of actual diabetic cases, which is an important clinical limitation.

Key insight: Glucose level is the single most powerful predictor of diabetes, followed by BMI and the Diabetes Pedigree Function (family history). These three features together drive most of the model's predictive power.

📈
Model Performance
Achieved 77.8% accuracy with 94% specificity. The 58% recall can be improved by adjusting the classification threshold, addressing class imbalance, or using ensemble models like Random Forest.
🔑
Key Predictors
Glucose, BMI, and Diabetes Pedigree Function are the top three most significant predictors. Blood Pressure and Skin Thickness had minimal influence and could potentially be dropped.
🚀
Future Improvements
Replace zero-value outliers with column medians, apply SMOTE for class balance, try XGBoost or Random Forest for higher accuracy, and deploy as a Shiny web app for real-time clinical use.
